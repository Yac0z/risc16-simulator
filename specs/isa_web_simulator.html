FLUX-16 · RISC-16

Instruction Set Architecture

16-bit RISC · 64 registers · 16 opcodes · 12 condition codes · 5-stage pipeline

Instruction Word Format

COND
[15:12] · 4 bits
OP
[11:8] · 4 bits
U I
[7:6] · 2 b
VAL
[5:0] · 6 bits
CONDCondition code — instruction only executes when this holds (T = always, default)
UUpdate-flags bit — 1 writes Z/N/C/V from the result (assembly suffix S)
OPOpcode — selects the operation (16 total)
IImmediate bit — 1: VAL is a signed 6-bit literal, 0: VAL is a register number
VAL6-bit field: register address R0–R63, or signed immediate −32..+31 (sign-extended to 16 bits by hardware)

Opcodes

CODE MNEM OPERATION DESCRIPTION
0x0AND ACC ← ACC AND op_bBitwise AND
0x1OR ACC ← ACC OR op_bBitwise OR
0x2XOR ACC ← ACC XOR op_bBitwise XOR
0x3NOT ACC ← ~ACCBitwise NOT — operand field is ignored
0x4ADD ACC ← ACC + op_bAdd — C = unsigned carry out, V = signed overflow
0x5SUB ACC ← ACC − op_bSubtract — C = 1 when no borrow (a ≥ b), V = signed overflow
0x6LSL ACC ← ACC << op_bLogical shift left — last shifted-out bit goes to C
0x7LSR ACC ← ACC >> op_bLogical shift right — zero-fills MSB, last shifted-out bit to C
0x8LDA ACC ← RAM[VAL]Load word from data memory at address VAL
0x9STA RAM[VAL] ← ACCStore ACC to data memory at address VAL
0xAMTA ACC ← op_bMove operand (immediate or register value) into ACC
0xBMTR R[VAL] ← ACCMove ACC into register R[VAL]
0xCJRP PC ← PC + op_bJump relative forward — PC + signed offset
0xDJRN PC ← PC − op_bJump relative backward — PC − offset (loop back)
0xEJPR PC ← R[VAL]Jump to absolute address held in register R[VAL]
0xFCAL RPC ← PC ; PC ← op_bCall subroutine — saves return address in RPC (R62), jumps to op_b
op_b = sign_ext(VAL) when I=1 (immediate mode) · op_b = R[VAL] when I=0 (register mode) · JRP/JRN use PC as ALU input A

Condition is evaluated at EXEC time against the current STATUS_REG values. A false condition suppresses the write-back but does not stall the pipeline.

Status Flags

Z  — Zero
Set when the ALU result equals 0x0000. Cleared otherwise. Affects Z and NZ conditions.
N  — Negative
Set when result bit 15 = 1 (two's-complement signed interpretation). Affects N, NN, P, NP.
C  — Carry
ADD: set on unsigned overflow past bit 15. SUB: set when a ≥ b (no borrow). Shifts: last bit shifted out. Affects C, NC.
V  — Overflow
Set when signed ADD or SUB overflows the two's-complement range (−32768 to +32767). Affects V, NV.
Flags are only updated when U = 1 in the instruction word (assembly suffix S). Without S the STATUS_REG is preserved.

Special Registers

R0 · ACC
Accumulator. Implicit destination of all ALU results (AND, OR, ADD, SUB, NOT, LSL, LSR, LDA, MTA). Also serves as ALU input A for all non-jump instructions.
R62 · RPC
Return Program Counter. Automatically written at FETCH1 with the address of the current instruction (before PC increments). Use JPRT r62 to return from a subroutine, or CALT addr to call one.
R63 · PC
Program Counter. Holds the address of the next instruction to fetch. Auto-incremented each FETCH1 stage. Written by JRP, JRN, JPR, and CAL during STORE to redirect control flow.
R1–R61 are general-purpose 16-bit registers. All 64 registers initialize to 0x0000 on reset.

Assembly Syntax

FORMAT — ONE INSTRUCTION PER LINE
OPCOND[S]  operand
COND is optional — defaults to T (always). S suffix sets U=1 (update flags). Semicolon starts a comment. Operand is rN for register N, a decimal integer, or 0xNN hex literal.
LOGIC & ARITHMETIC
ANDTS r3; ACC &= R3, upd flags
ADDTS r1; ACC += R1, upd flags
SUBTS 1 ; ACC -= 1 (imm)
LSLTS 4 ; ACC <<= 4
NOTTS r0; ACC = ~ACC
LOAD / STORE / MOVE
MTATS 0x1F; ACC = 0x1F
MTRT r2 ; R2 = ACC
STAT 0x08; RAM[8] = ACC
LDATS 0x08; ACC = RAM[8]
CONDITIONAL EXECUTION
ADDTS r2; always (T)
SUBZS r3; only if Z=1
ADDNZSr4; only if Z=0
ANDNS r5; only if N=1
ADDCS r6; only if C=1
CONTROL FLOW
CALT 0x08; call addr 8
JPRT r62 ; return (PC=RPC)
JRPT 3 ; jump forward 3
JRNT 2 ; jump back 2
JRNNZ 1 ; back 1 if NZ
